# 동서대학교 좌석 및 기자재 관리 시스템 (Seat & Equipment Management System) 주요 기능 정의서

본 문서는 동서대학교 좌석 및 기자재 관리 시스템(WPF 기반 데스크톱 애플리케이션)의 모든 주요 기능과 비즈니스 로직을 체계적으로 정리한 기능 정의서입니다.

---

## 1. 대시보드 (Dashboard) 

시스템의 메인 화면으로, 실시간 모니터링 뱃지와 좌석 배치도, 그리고 간단한 메모장 기능을 통합 제공합니다.

### 1.1. 실시간 시스템 정보 및 상태 갱신

* **실시간 날짜/시간**: 현재 시간 및 요일 정보를 헤더에 상시 표시합니다.



### 1.2. 경고/알림 뱃지 시스템

* **상상Lab 승인 대기 뱃지**: 대기 중인 상상Lab 신청 건수를 빨간색 카드로 헤더에 노출합니다. 클릭 시 상상Lab 승인 탭으로 즉시 이동합니다.
* **캐비닛 승인 대기 뱃지**: 대기 중인 캐비닛 신청 건수를 주황색 카드로 헤더에 노출합니다. 클릭 시 캐비닛 현황 탭으로 즉시 이동합니다.
* **기자재 반납 연체 뱃지**: 대여 기한이 지난 미반납 기자재 건수를 빨간색 카드로 노출합니다. 클릭 시 기자재 현황 탭으로 즉시 이동합니다.

### 1.3. 좌석 배치도 (Seat Layout Grid)

* **그리드 레이아웃**: 52개의 좌석과 물리적 이동 경로 및 기둥(Pillar) 구조를 시각화한 그리드를 제공합니다.
* **연도 및 학기별 배치 이력 관리**:
  * 사용자가 선택한 연도 및 학기(예: 2026년 여름방학, 1학기 등) 데이터를 로컬 세션 캐시(`_seatLayoutCache`)에 저장하여, 학기 전환 시 배치 정보가 유실되지 않도록 보존합니다.
* **개별 좌석 카드 조작**:
  * **학생 상세 정보 모달**: 좌석 카드 클릭 시 배정된 학생의 학적 정보(학번, 이름, 소속, 지도교수, 이메일) 및 3일간의 출석 현황(출석/결석/지각) 목록을 팝업 형태로 조회합니다.
  * **학생 신규 배정**: 배정되지 않은 빈 좌석 카드를 클릭하면 직접 학생의 상세 정보를 입력하여 좌석에 배치할 수 있습니다.
* **좌석 고정 모드 (Seat Fix Mode)**:
  * 특정 좌석들을 다중 선택하여 '고정' 상태로 설정(🔒 아이콘 표시 및 주황색 테두리 하이라이트)할 수 있습니다. 고정된 좌석은 랜덤 배정 시 대상에서 제외됩니다.
* **제약 조건 기반 랜덤 배정 (Random Allocation)**:
  * 고정되지 않은 좌석의 학생 데이터만 무작위로 섞어 재배정합니다.
  * **수평 거리 제약 조건**: 동일한 행(Row)으로 재배치될 경우, 이전 좌석 위치로부터 수평 거리가 최소 2칸 이상 떨어지도록 알고리즘 제약 조건을 적용합니다. (1000회 시도 내 제약 조건 충족 실패 시, 사용자 안내 후 기본 셔플로 대체 실행)
* **좌석 데이터 삭제**:
  * **선택 삭제 모드**: 배치도 내에서 삭제를 원하는 좌석만 다중 선택하여 인적사항 및 출석 데이터를 일괄 삭제합니다.
  * **전체 삭제**: 모든 좌석의 인적 사항 및 출석 데이터만 일괄 초기화합니다 (좌석 번호와 고정 여부는 유지).
* **데이터 새로고침**: 마스터 데이터베이스 정보를 기준으로 고정되지 않은 좌석의 배치 정보를 최신 학생 데이터로 다시 덮어써 초기화합니다.

### 1.4. 대시보드 메모장 (Memo/Todo)

* **실시간 메모 추가**: 우측 사이드 패널에 텍스트를 입력하여 즉시 공지사항이나 할 일을 추가합니다.
* **메모 수정**: 등록된 메모의 수정 단추를 클릭해 텍스트 입력창으로 다시 불러와 수정 반영합니다.
* **메모 삭제**: 해결된 일감이나 불필요해진 메모를 즉시 목록에서 제거합니다.

## 2. 기자재 현황 관리 (Equipment Management)

학교 내 연구 및 실험 장비의 대여 및 반납 상태를 통합 관리합니다.

* **통계 카드**: 사용 가능한 총 기자재 수, 대여 중인 기자재 수, 승인 대기 중인 기자재 신청 수에 대한 실시간 통계를 상단에 제공합니다.
* **기자재 대여 신청 승인**: 구글 폼 등을 통해 신청된 기자재 대여 요청 정보(신청일, 학생 정보, 장비명) 리스트를 확인하고 승인/반려 처리를 수행합니다.
* **실시간 대여 현황 테이블**: 현재 대여 중인 기자재의 학생명, 장비명, 대여 시작일, 대여 기한(기본 7일), 반납 완료 상태를 관리합니다.
  * **연체 표시**: 대여 기한이 초과된 항목은 자동으로 '연체' 상태로 분류되어 빨간색 상태 표시등이 점등됩니다.
* **신속 반납 및 실행 취소 (Undo Stack)**:
  * 대여 중인 항목에 대해 '반납' 처리 시 대여 목록에서 즉시 제거됩니다.
  * 실수로 반납을 처리한 경우, LIFO(후입선출) 방식의 실행 취소 스택(`_rentalUndoStack`)을 활용해 '되돌리기' 버튼 클릭 시 직전 반납 건을 대여 목록으로 즉시 복구합니다.
* **기자재 관리 보조 API**: 데이터 삭제, 고정, 외부 추출(Export) 기능 확장을 위한 이벤트 스터브가 마련되어 있습니다.

## 3. 캐비닛 현황 관리 (Cabinet Management)

학생들이 사용하는 보관함의 배정 상태를 직관적인 그리드 뷰로 시각화하여 제공합니다.

* **캐비닛 대기 신청 승인**: 캐비닛 배정을 신청한 학생의 목록을 확인하고 승인/반려합니다.
  * **자동 좌석 배치 연동**: 캐비닛 사용 승인을 완료하면, 대시보드의 첫 번째 빈 좌석을 탐색하여 해당 학생의 학적 데이터를 대시보드 좌석 배치에 자동으로 연동 추가합니다.
* **블록형 캐비닛 배치도 (Cabinet Block View)**:
  * **구조**: Block 1(1~24번), Block 2(25~48번)로 나누어 총 48개의 캐비닛 구조를 시각화합니다.
  * **캐비닛 셀 상태 시각화**: 사용 중인 캐비닛은 부드러운 파란색 배경으로 강조 표시되며, 배정된 학생의 학번/이름 및 사용 기간(예: 07/16 ~ 08/16)이 셀 내에 상세 표시됩니다. 빈 캐비닛은 기본 상태로 표시됩니다.
* **캐비닛 관리 보조 API**: 캐비닛 삭제, 고정, 데이터 내보내기 기능 확장을 위한 인터페이스가 마련되어 있습니다.

## 4. 상상Lab 승인 관리 (SangsangLab Approval)

협업 및 실험 공간인 상상Lab의 이용 권한 신청을 구글 폼과 연동하여 승인합니다.

* **신청 정보 카드 리스트**: 구글 폼으로 접수된 상상Lab 사용 신청서를 학생 정보 위주의 세련된 카드로 나열합니다.
* **상세 정보 모달 조회**: 신청 카드 우측의 상세 조회 버튼을 통해, 신청 학생의 이메일 및 세부 인적 사항 정보를 좌석 상세 모달과 동일한 포맷으로 팝업 조회할 수 있습니다.
* **원클릭 승인/반려**:
  * **승인**: 신청 건을 승인 완료 처리하고, 대시보드의 비어있는 좌석 카드를 찾아 해당 학생을 자동으로 착석시킵니다. 착석과 동시에 당일 기준 출석 기록("출석")이 자동으로 생성됩니다.
  * **반려**: 대기 목록에서 해당 요청을 즉시 제거합니다.

## 5. 데이터 관리 (Data Management - Master Database)

외부 입력(예: Google Forms 전체 누적 데이터)을 모방한 원천 학생 마스터 데이터베이스를 직접 관리합니다.

* **학생 데이터 검색 및 목록 조회**: 데이터 관리 탭 내 마스터 그리드(`GridMasterStudents`)를 통해 등록된 전체 학생 리스트를 조회하고 개별 학생의 출석부 세부 정보 등을 확인합니다.
* **학적 정보 직접 수정**: 그리드에서 특정 학생을 선택하면 우측 편집 패널이 활성화되어 이름, 소속 학과, 지도교수, 이메일 정보를 실시간 수정하고 즉시 데이터베이스에 반영할 수 있습니다.
* **마스터 데이터 삭제**: 선택된 학생 데이터를 마스터 DB에서 영구 삭제합니다. (이때, 이미 대시보드 좌석표에 배치된 데이터는 딥 카피(`Clone()`)된 상태이므로 좌석표 데이터의 안정성이 확보됩니다.)

## 6. 개발 및 테스트 편의 기능

개발 과정 및 실무 운영 시 테스트 효율성을 높이기 위한 유틸리티 기능입니다.

* **구글 폼 신청 시뮬레이터 (Google Form Simulator)**:
  * '구글 폼 신청 시뮬레이션' 버튼을 누르면 랜덤한 학생 이름, 학번, 학과 및 신청 구분(상상Lab / 캐비닛)을 가지는 가상의 신청서가 실시간으로 자동 접수됩니다.
  * 접수 즉시 헤더의 경고/알림 뱃지와 대기 건수 상태 정보가 실시간으로 올라가 승인/반려 로직 테스트가 용이합니다.
* **실시간 변경 감시 및 자동 빌드/실행 도구 (`auto-build.ps1` & `auto-build.bat`)**:
  * 개발 디렉토리 내부의 소스 파일 변경(`.cs`, `.xaml`, `.xaml.cs`)을 실시간으로 감시하는 감시자(Watcher) 스크립트입니다.
  * 변경 감지 시 자동으로 기존 앱 프로세스를 안전하게 종료하고, `dotnet build`를 실행하여 컴파일한 뒤, 새로운 빌드를 즉시 실행시키는 Hot Reload 방식의 자동화 스크립트를 내장하고 있습니다.
